# F27 P10 — EDOG Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the EDOG-side contract surface (catalog assembler + typed schema generator + pipeline rewiring + three-column workbench + executor extension + telemetry + revision-snapshot flags) that consumes the FLT `flt/qa-contract-v1.0` tag.

**Architecture:** Sana owns the contract shape and cross-repo boundaries; Vex implements the backend C#/Python contract path; Pixel owns the workbench and catalog-health UI; Sentinel owns every test gate and vetoes any legacy opaque-JSON escape hatch. The implementation lands as a big-bang model migration: first replace the shared QA models and options surface, then add the catalog and schema stack, then rewire orchestration/execution/telemetry, then rewrite the frontend against the new typed contract.

**Tech Stack:** C# DevMode files under `src/backend/DevMode/`, Python proxy/catalog scripts, vanilla JS + existing `qa-panel.css`, JSON Schema strict mode, Azure OpenAI `/chat/completions` + `/responses`, pytest, Node `node:test`, and the existing `tests/dotnet/EdogQaE2E.Tests` harness.

**Depends on:** FLT Phase 0 plan complete and `flt/qa-contract-v1.0` tag available.

---

## File Structure

`src/backend/DevMode/EdogQaModels.cs` is the breaking contract root. It owns the renamed stimulus kinds, new verdict states, new matcher assertion vocabulary, typed matcher-value union, and per-scenario catalog-hash envelope that every downstream backend and frontend surface will consume.

`src/backend/DevMode/EdogQaFeatureFlags.cs` is removed from the live path and replaced by `src/backend/DevMode/IQaContractOptionsProvider.cs`. The new interface and implementation own revisioned immutable contract snapshots so generation, execution, and UI reads stay consistent across config reloads.

`src/backend/DevMode/EdogQaTelemetryRedactor.cs` is a new helper that centralizes hashing, truncation, reason-code normalization, and sampling decisions for the contract telemetry stream. `src/backend/DevMode/EdogQaTelemetry.cs` becomes a thin event-emitter layer on top of it.

`src/backend/DevMode/EdogQaContractCatalog.cs` is the new assembler for the per-zone contract catalog. It owns provider orchestration, required-provider hard-fail logic, the immutable `CatalogSnapshot` envelope, canonical hashing, and assembler-side cap assertions.

`src/backend/DevMode/EdogQaDagScanner.cs` is the Roslyn scanner for `[DagDefinition]` DAG classes. It converts FLT DAG declarations into stable `DagTrigger` slot descriptors with capture topics and purpose text.

`src/backend/DevMode/EdogQaFileTimerScanner.cs` is the Roslyn scanner for `[EdogFileEventSeam]` and `[EdogTimerSeam]` surfaces. It emits `FileEvent` and `TimerTick` slots from the actual FLT seam attributes rather than guessed DI abstractions.

`src/backend/DevMode/EdogQaDiRegistryProvider.cs` remains the DI-slot provider entry point, but now filters against FLT’s new `[EdogDirectInvokeSeam]`/`IQaDirectInvokeRegistry` surface and reports `ok`/`degraded`/`empty`/`failed` instead of silently treating empty catalogs as normal.

`src/backend/DevMode/EdogQaOmniSharpProvider.cs` remains the topic-field discovery layer, but expands its topic-router scan to include anonymous-type member names so matcher `topic.field` enums come from real router shapes.

`src/backend/DevMode/EdogQaLlmClient.cs` becomes the typed-schema generator. It owns the full Editor schema, `PartialRepairSchema`, `SingleScenarioSchema`, generator-side cap rechecks, P1/P5/P6/P7 prompt machinery, and the editor/architect request settings that must honor the strict contract.

`src/backend/DevMode/EdogQaCapabilityProbe.cs` remains the model-capability surface but adds role-specific latency fallback, `reasoning_effort: medium` for Architect, and the hook points needed for per-role temperature/reasoning config.

`src/backend/DevMode/EdogQaScenarioProjector.cs` shrinks into a straight typed projection layer. Its only job after Phase 1 is to preserve audit fields while copying already-typed stimulus/matcher objects into engine-ready records.

`src/backend/DevMode/EdogQaScenarioValidator.cs` remains the semantic backstop. It keeps the existing gates, adds the new `MatcherTypeConsistencyGate`, extends grounding with the active slot-match check, and verifies slot/topic consistency that strict mode cannot express.

`src/backend/DevMode/EdogQaScenarioOrchestrator.cs` stays the zone driver. It now captures the options revision snapshot, requests a catalog per zone, uses the capped reachability formula, runs the richer repair loop, and owns the single-scenario attempt-3 escalation path.

`src/backend/DevMode/EdogQaStimulusDispatcher.cs` becomes the EDOG-side client for the new FLT dispatch endpoints. It owns sync dispatch for HTTP/DI/SignalR/FileEvent/TimerTick, async polling for DAG, cancellation propagation, capability checks, and timeout/error-code normalization.

`src/backend/DevMode/EdogQaExecutionEngine.cs` compares per-slot and per-topic hashes against the live catalog snapshot, blocks stale scenarios, routes non-HTTP kinds through the dispatcher, and surfaces `stale`/`inconclusive` verdicts without collapsing them into `failed`.

`src/backend/DevMode/EdogQaAssertionEngine.cs` is a breaking rewrite. It removes regex-based assertions entirely and evaluates the seven new matcher assertions against the typed matcher-value union.

`src/backend/DevMode/EdogQaScenarioLinter.cs` remains the defense-in-depth linter. It keeps LNT001/LNT003/LNT004 at `Error` and adds LNT011 so any format-string or regex-like matcher value still gets blocked loudly.

`src/backend/DevMode/EdogQaRunStore.cs` owns the on-disk JSON envelope migration. It quarantines all pre-`p10` scenarios in-place inside `QaRunRecord` entries and persists the new verdict and catalog-hash states.

`src/backend/DevMode/edog.py` must be updated because new DevMode C# files do not ship to FLT unless they are registered in `DEVMODE_FILES`. Even though the spec appendix omits it, the repo’s actual deployment plumbing requires explicit registration for every new backend file.

`tests/dotnet/EdogQaE2E.Tests/LlmClientHarness.cs` remains the behavioral harness for Architect/Editor strict-schema generation. It is extended to cover typed `stimulus`, typed `matchers`, nullable unions, few-shot enablement, and repair-schema shapes.

`tests/dotnet/EdogQaE2E.Tests/ValidatorHarness.cs` remains the semantic validator harness. It gains cases for matcher-type consistency, grounding-slot mismatch, topic-field consistency, and new stale/inconclusive outcome paths.

`tests/dotnet/EdogQaE2E.Tests/OrchestratorHarness.cs` remains the orchestration harness. It gains cases for revision snapshots, capped reachability, partial-repair splice, and attempt-3 single-scenario escalation.

`tests/dotnet/EdogQaE2E.Tests/CapabilityProbeHarness.cs` remains the probe harness. It adds cases for 15-second first-token timeout, 60-second total timeout, per-role reasoning effort, and architect fallback telemetry.

`tests/dotnet/EdogQaE2E.Tests/BroadcastProjectionHarness.cs` remains the round-trip harness for wire projection and must be updated to the renamed stimulus kinds and typed matcher payloads.

`tests/dotnet/EdogQaE2E.Tests/ApiSurfaceRenderHarness.cs` remains the backend surface renderer and now exposes catalog snapshot fields and new outcome badges used by the analysis/curation UI.

`tests/dotnet/EdogQaE2E.Tests/Program.cs` must expose any new harness subcommands needed for contract-catalog, execution, telemetry, and live-dispatch gate coverage.

`tests/test_qa_prompt_shape.py` remains the source-grep guard on LLM prompt/system-message structure. It must be updated from opaque `stimulusSpec`/`matcherSpec` language to typed `stimulus`/`matchers` contract language and the renamed kinds.

`tests/test_qa_e2e.py` remains the Python wrapper around the .NET harness. It is extended with new assertions for schema, catalog, replay, migration, disable, telemetry, and capability-skew gates.

`tests/test_qa_execution_wiring.py` remains the fast source-grep contract for hub/engine wiring. It must move from “engine called” checks to “catalog hash / dispatcher / stale / inconclusive / capability” checks where those guarantees are structural.

`tests/test_qa_telemetry.py` remains the source-grep and structural telemetry contract. It gains the contract-health events from §7.3 plus redaction helper wiring.

`tests/test_qa_history_store.py` remains the run-store structural contract. It gains the `p10` migration and the new verdict/catalog-hash fields.

`tests/test_qa_capabilities.py` remains the capability and honesty-gate suite. It adds `GET /devmode/qa/capabilities` proxy behavior and `FLT_VERSION_MISMATCH` fail-fast checks.

`tests/test_playground_dispatch.py` and `tests/test_playground_dispatch_validators.py` remain the best place to pin proxy and dispatch-client request/response shapes. They must be extended for sync/async contract dispatch and the control-token / timeout / 64KB validation paths.

`tests/test_framework_endpoints_loader.py` and `tests/test_framework_endpoints_lint.py` continue guarding `data/framework-endpoints.json`. They must cover any SignalR registry additions required by the contract slot provider.

`tests/js/test-qa-contract-left-column.mjs`, `tests/js/test-qa-contract-middle-column.mjs`, and `tests/js/test-qa-contract-right-column.mjs` are new Node `node:test` suites. They pin the three independent workbench chunks that Pixel must land without reintroducing the legacy opaque authoring path.

`src/frontend/js/qa-analysis.js` remains the analysis-stage controller, but grows the catalog-health strip for snapshot metadata and provider status pills.

`src/frontend/js/qa-curation.js` is rewritten in place into the three-column workbench. It owns scenario-list filtering, slot selection, typed parameter entry, matcher composition, issues, and inline last-run results with no JSON-text escape hatch.

`src/frontend/css/qa-panel.css` remains the shared QA panel stylesheet. It must absorb the catalog-health strip, the three-column workbench layout, the slot-picker rich labels, typed matcher controls, issues strip, and last-run strip styling using the established tokens.

`scripts/dev-server.py` remains the local proxy and gains `/api/contract/catalog/{zoneId}` plus `/api/contract/capabilities`. It also needs request validation and error translation that preserve the contract surface without leaking the control token.

`scripts/flt_catalog.py` remains the framework-endpoint extraction helper. It is extended so the SignalR slot provider can consume the existing hand-curated registry in contract-catalog shape.

`config/edog-config.template.json` must define the new `qa.contract.*` defaults, including the contract kill switch, per-kind disable list, few-shot toggle, and control token placeholder.

`data/framework-endpoints.json` stays the hand-curated framework registry and must include any additional SignalR contract slot entries that the new provider depends on.
