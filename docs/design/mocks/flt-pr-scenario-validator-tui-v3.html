FLT PR Scenario Validator · presentation contract v3

What the run prints

A Copilot CLI skill that validates an FLT pull request against a live environment. It can only print plain Markdown + Unicode marks into the terminal — no colour control, no live redraw. So the design is all hierarchy, restraint, and plain English: a smart engineer who has never seen FLT should understand every line.

✓good ✕broken by the change ▲needs attention ◇side note ◌never ran ▣locked target ▸doing / asking ◆headline / box Every mark is paired with a word — colour is never the only signal. Two domain words are allowed, glossed once: DAG = the chain of steps FLT runs in order; MLV = a managed table FLT keeps up to date.
