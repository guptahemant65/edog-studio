## F08: TOKEN INSPECTOR TAB

### What Tokens Exist in the FLT Service (from codebase research)

| Token Type | Generated By | Used For | Auth Header | Interceptable? |
|---|---|---|---|---|
| **MWC V1 User** | `BaseTokenProvider` → `ITridentAuthorityProvider.GetServiceMwcToken()` | OneLake file operations (DAG metrics, locks, settings) | `FMVTokenCredential` (Azure SDK) | ✅ Via DelegatingHandler pipeline |
| **MWC V1 for GTS/Spark** | `GTSBasedSparkClient.GenerateMWCV1TokenForGTSWorkloadAsync()` | Spark SQL/PySpark execution | `Authorization: MwcToken {token}` | ⚠️ Direct header injection — need HttpClient hook |
| **S2S (OneLake Storage)** | `S2STokenProvider.GetS2STokenForOneLakeAsync()` | Private Link OneLake writes | `x-ms-s2s-actor-authorization: Bearer {token}` | ✅ Via `OneLakeRequestTracingHandler` |
| **S2S (MWC Rollout)** | `S2STokenProvider.GetS2STokenForMWCRolloutAsync()` | MWC rollout operations | `x-ms-s2s-actor-authorization` | ✅ Via handler pipeline |
| **S2S (PBI Shared)** | `S2STokenProvider.GetS2STokenForPbiSharedAsync()` | PBI Shared API calls | `Authorization: Bearer` + `x-ms-s2s-actor-authorization` | ⚠️ PBIHttpClientFactory direct |
| **OBO (User)** | `HttpTokenUtils.GetOboTokenAsync()` | OneLake REST API on behalf of user | `Authorization: Bearer {oboToken}` | ✅ Via handler pipeline |
| **Bearer (AAD)** | External (Playwright auth) | Fabric public APIs (redirect host) | `Authorization: Bearer {aadToken}` | ✅ We generate this ourselves |
| **MWC (EDOG-generated)** | `POST /metadata/v201606/generatemwctoken` | Capacity host APIs (tables, DAG) | `Authorization: MwcToken {token}` | ✅ We generate this ourselves |

### Interception Strategy

**Two-pronged capture:**

1. **DelegatingHandler** (captures pipeline-based tokens):
   ```csharp
   public class EdogTokenInterceptor : DelegatingHandler
   {
       protected override async Task<HttpResponseMessage> SendAsync(
           HttpRequestMessage request, CancellationToken ct)
       {
           // Capture Authorization header
           var auth = request.Headers.Authorization;
           // Capture S2S header
           var s2s = request.Headers.TryGetValues("x-ms-s2s-actor-authorization", out var s2sValues)
               ? s2sValues.First() : null;
           // Capture client-authorization
           var clientAuth = request.Headers.TryGetValues("x-ms-client-authorization", out var caValues)
               ? caValues.First() : null;

           // Stream to EDOG via WebSocket
           EdogLogServer.BroadcastToken(new TokenCapture {
               Scheme = auth?.Scheme,          // "Bearer", "MwcToken"
               TokenPreview = auth?.Parameter?[..20] + "...",  // First 20 chars only (security)
               TargetUrl = request.RequestUri?.ToString(),
               S2SPresent = s2s != null,
               ClientAuthPresent = clientAuth != null,
               Timestamp = DateTime.UtcNow,
               // Decode JWT claims WITHOUT logging the full token
               Claims = DecodeJwtClaims(auth?.Parameter),
           });

           return await base.SendAsync(request, ct);
       }
   }
   ```

2. **Direct capture hooks** (for GTSBasedSparkClient, OneLakeRestClient):
   - Override `SendHttpRequestAsync()` in our `EdogTracingSparkClientFactory` (already planned for F14 Spark Inspector)
   - The same interceptor captures both Spark requests AND the tokens used

```json
  "source": "EdogTokenInterceptor|GTSBasedSparkClient|OneLakeRestClient|EDOG_Proxy",
  "scopeKey": "ws:lh:cap tuple or service name",
  "ttlSeconds": 2520,
  "claimsPreview": { "aud": "...", "exp": 1775740000, "oid": "..." },
  "timestamp": "2026-04-09T14:22:03Z"
}
```

**Security:** NEVER stream full token strings over WebSocket. Only: scheme, audience, TTL, claims preview, source class. The [Copy] button fetches the full token on demand from the server.

## F09: SYSTEM FILES EXPLORER TAB

### What Files the FLT Service Operates On (from codebase research)

All via `OnelakeBasedFileSystem` → `DataLakeDirectoryClient` → OneLake DFS.

| File Category | Path Pattern | Operations | Trigger |
|---|---|---|---|
| **DAG definition** | `DagExecutionMetrics/{iterationId}/dag.json` | Create, Read | DAG start |
| **DAG metrics** | `DagExecutionMetrics/{iterationId}/dag_metrics.json` | Create, Update | DAG completion |
| **Node metrics** | `DagExecutionMetrics/{iterationId}/node_{nodeId}_metrics.json` | Create | Per-node completion |
| **Index files** | `Index_StartTime_{id}/{reverseEpoch}_{iterationId}_{status}` | Create, Rename (status change) | DAG start, DAG terminal |
| **Execution locks** | `DagExecutionMetrics/{artifactId}.lock` | Create, Delete | DAG start, DAG end |
| **Schedule locks** | `DagExecutionMetrics/{mlvDefId}.lock` | Create, Delete | Scheduled DAG start/end |
| **DAG settings** | `dagsettings.json` + `.lock` | Read, Write | Settings change |
| **Environment** | `environment.json` + `.lock` | Read, Write | Environment attach |
| **MLV definitions** | `MLVExecutionDefinitions/{defId}.json` | Create, Update | MLV definition CRUD |
| **Recovery files** | `MLVExecutionDefinitions/{defId}-recovery.json` | Create | On failure |
| **DQ report** | `{DQBase}/dqReport.json` + `.lock` | Create, Update | DQ execution |
| **Marker files** | `_mlv_system_sys_*.marker` | Create | First metrics write |

### Interception Strategy

Wrap `IFileSystem` with a decorator:

```csharp
public class EdogFileSystemInterceptor : IFileSystem
{
    private readonly IFileSystem _inner;

    public async Task CreateOrUpdateFileAsync(string path, string content, TimeSpan ttl, CancellationToken ct)
    {
        // Stream to EDOG BEFORE the operation
        EdogLogServer.BroadcastFileOp(new FileOperation {
            Op = "WRITE",
            Path = path,
            ContentSize = content.Length,
            ContentPreview = content[..Math.Min(500, content.Length)],  // First 500 chars
            HasTTL = ttl != default,
            TTL = ttl,
            Timestamp = DateTime.UtcNow,
        });

        await _inner.CreateOrUpdateFileAsync(path, content, ttl, ct);
    }

    // Same pattern for: CreateDirIfNotExists, CreateEmptyFileIfNotExists,
    // ReadFileAsync, DeleteFileAsync, ListAsync, RenameAsync, ExistsAsync
}
```

Register via DI: Replace `IFileSystemFactory` → `EdogFileSystemFactory` that wraps the real `OnelakeFileSystemFactory`.

## Security Considerations

- **NEVER stream full token strings** over WebSocket — only scheme, audience, TTL, claims preview
- **Content preview** for file operations capped at 500 chars — enough to see structure, not enough to leak sensitive data
- **Lock force-unlock** requires confirmation dialog
- **All data stays on localhost** — no external transmission
