# EDOG Studio — Design Context

## Target Audience
Senior C# backend engineers at Microsoft developing FabricLiveTable (FLT). They live in Visual Studio, terminals, and Azure Data Explorer. They use this tool 8+ hours/day. They value information density, keyboard shortcuts, and zero friction over aesthetics.

## Use Cases
- Debug failing DAG executions by reading real-time logs and telemetry
- Browse workspaces/lakehouses/tables and deploy to a target lakehouse
- Run and cancel DAG executions, inspect Spark HTTP traffic
- Manage feature flags (view rollout state, override locally, create PRs)
- Test Fabric API calls with pre-configured endpoints
- Monitor token health, lock state, service status

## Brand Personality / Tone
**Surgical precision. Calm authority.** The feeling of a well-organized control room where everything is exactly where you expect. Not flashy — confident. Dense but readable. Fast but not bare. Every element earns its place.

## Technical Constraints
- Single HTML file with inline CSS/JS (no React/Vue/Angular)
- Must handle 10,000+ log entries with virtual scrolling
- Real-time WebSocket data streaming at 150ms batches
- Dark theme primary
- Keyboard-first (Ctrl+K command palette, number keys for tabs)

## Design Principles
1. **Zero context switches** — everything a dev needs without leaving this tab
2. **Information density over decoration** — pack more data, not more chrome
3. **Keyboard-first** — if you reach for the mouse, the design failed
4. **Progressive disclosure** — start simple, reveal depth on interaction
5. **Instant or bust** — if the user perceives lag, it's broken

## Anti-Patterns (Never Do)
- No emoji as icons — use Unicode symbols or inline SVG
- No glassmorphism or glow effects
- No gradient text
- No cards wrapping cards
- No modals when a drawer or inline expansion works
- No pure black (#000) or pure white (#fff) backgrounds
- No HSL — use OKLCH for perceptually uniform colors
